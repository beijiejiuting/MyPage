    # HTTP会话

    用户打开浏览器，访问wb服务器的资源，会话建立，直到有一方断开连接，会话结束。在一次会话中可以包含多次请求和响应

    # 会话跟踪

    服务器需要识别多次请求是否来自于同一浏览器，以便在同一次会话的多次请求间共享数据, 维护浏览器状态

    # 会话跟踪方案
    # 客户端会话跟踪技术：Cookie

    服务器响应给客户端一个cookie,保存在客户端,此次会话中,客户端每次请求都会携带cookie

    # 服务端会话跟踪技术：Session

    服务器保存一些session,使用cookie传递sessionID,通过id获取session内容

    # 令牌技术

    服务端传递令牌给客户端,客户端每次请求携带令牌

    JWT令牌(JsonWebToken)定义了一种简洁的、自包含的格式，用于在通信双方以json数据格式安全的传输信息。由于数字签名的存在，这些信息是可靠的。

    第一部分：Header((头)，记录令牌类型、签名算法等。例如：{"alg":"HS256","type":"JWT") 第二部分：Payload(有效载荷)，携带一些自定义信息、默认信息等。例如："id":"1","username":"Tom"門 第三部分：Signature(签名)，防止Token被篡改、确保安全性。将header、payload,并加入指定秘钥，通过指定签名算法计算而来。

    Base64编码头和载荷

    # SpringAOP

    Aspect Oriented Programming面向切面编程

    @Aspect AOP类 @Around/Before/After/AfterReturning/AfterThrowing ("切入点表达式")

    pre joinPoint.proceed(); after

    连接点：JoinPoint,可以被AOP控制的方法（暗含方法执行时的相关信息） 通知：Advice,指哪些重复的逻辑，也就是共性功能（最终体现为一个方法） 切入点：PointCut,匹配连接点的条件，通知仅会在切入点方法执行时被应用 切面：Aspect,描述通知与切入点的对应关系（通知+切入点） 目标对象：Target,通知所应用的对象

    # Bean管理

    Bean的默认名字: 类->类名小写 方法->方法名

    把一个类@Component,交给IOC容器管理

    # 获取Bean

    默认情况下，Spring项目启动时，会把bean都创建好放在IOC容器中，如果想要主动获取这些bean,可以通过如下方式：

    # 根据name获取bean:

    Object getBean(String name)

    # 根据类型获取bean:
    T getBean(ClassrequiredType)
    # 根据name获取bean(带类型转换)：
    T getBean(String name,ClassrequiredType)

    # Bean的作用域

    通过@Scope配置

    Spring支持五种作用域，后三种在web环境才生效： singleton 容器内同名称的bean只有个实例（单例）（默认） prototype 每次使用该bean时会创建新的实例（非单例） request/session/application 每个请求/会话/应用范围内会创建新的实例

    延迟初始化

    @Lazy第一次使用时才实例化

    # 第三方Bean

    如果要管理的bean对象来自于第三方（不是自定义的），是无法用@Comporent及衍生注解声明bean的，就需要用到@Bean注解。

    # 通过方法返回

    写一个配置类@Configuration,在配置类中写方法,返回需要的bean对象,对此方法@Bean,交给IOC容器

    通过@Bean注解的name或value属性可以声明bean的名称，如果不指定，默认bean的名称就是方法名。 如果第三方bean需要依赖其它bean对象，直接在bean定义方法中设置形参即可，容器会根据类型自动装配
